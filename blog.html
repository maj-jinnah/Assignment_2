Var - Variables declared with var are function-scoped. But if a variable is declared outside of any function, it becomes a global variable. Variables declared with var are hoisted up to the top of their scope during the compilation phase. Also, it can be reassigned and re-declared.
Let - Variables declared with let are block-scoped. It means it is only visible within the block where they are defined. Unlike var, let is not hoisted. It can be re-assigned but not re-declared.
Const - Variables declared with const also have block scope. Const has to be assigned value before its use and once a value is assigned to a const variable, it cannot be reassigned or er declared. Const variables are not hoisted up.

undefined:undefined is a primitive value automatically assigned When a variable is declared but not assigned any value. It is a default return value from a function when it doesn't explicitly return anything.
null: null is a special value that absence of any object value.
